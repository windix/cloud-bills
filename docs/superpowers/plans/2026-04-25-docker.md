# Docker + Config Folder Reorganisation Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Package the cloud-bills backend and Vue dashboard into a single Docker image with credentials mounted as a volume at runtime.

**Architecture:** Multi-stage Dockerfile — Stage 1 builds the Vue dashboard with Vite (oven/bun), Stage 2 is the lean runtime (oven/bun) that runs the Hono backend and serves the dashboard's `dist/` as static files. Provider config YAML files and GCP keys live in `config/` and are volume-mounted at runtime.

**Tech Stack:** Bun, Hono `serveStatic`, Vite (dashboard build), Docker multi-stage builds, docker-compose.

## Task 3: Add static file serving to Hono

**Files:**
- Modify: `src/index.ts`

The `serveStatic` middleware from `hono/bun` serves files from a root directory. When a request comes in for a path that doesn't match any file on disk, it calls `next()`. The second `serveStatic` call (the SPA fallback) always rewrites the path to `index.html`, so any unmatched route returns the Vue app entry point.

Both middleware calls go **after** all API routes so API paths are never swallowed by the static handler.

- [ ] **Step 1: Add `serveStatic` import to `src/index.ts`**

At the top of the file, after the existing imports, add:

```typescript
import { serveStatic } from "hono/bun";
```

- [ ] **Step 2: Add static middleware at the bottom of `src/index.ts`**

Before the `export default { port: 3000, fetch: app.fetch }` line, add:

```typescript
// Serve built dashboard — must come after all API routes
app.use("/*", serveStatic({ root: "./dashboard/dist" }));
// SPA fallback: any unmatched path returns index.html for client-side routing
app.use("/*", serveStatic({ root: "./dashboard/dist", rewriteRequestPath: () => "index.html" }));
```

- [ ] **Step 3: Run existing tests**

```bash
bun test src/index.test.ts
```

Expected: Both tests pass. The static middleware only fires after API routes, so `/openapi.json` and `/docs` are unaffected. In the test environment there is no `dashboard/dist/` on disk, so the static handler calls `next()` and Hono returns 404 for unknown paths — which is correct.

- [ ] **Step 4: Commit**

```bash
git add src/index.ts
git commit -m "feat: serve dashboard static files from Hono"
```

## Task 4: Create Dockerfile and .dockerignore

**Files:**
- Create: `Dockerfile`
- Create: `.dockerignore`

- [ ] **Step 1: Create `Dockerfile`**

```dockerfile
# Stage 1: build the Vue dashboard
FROM oven/bun AS dashboard-builder
WORKDIR /build
COPY dashboard/package.json dashboard/bun.lock ./
RUN bun install --frozen-lockfile
COPY dashboard/ .
RUN bunx vite build

# Stage 2: runtime
FROM oven/bun AS runtime
WORKDIR /app
COPY package.json bun.lock ./
RUN bun install --frozen-lockfile --production
COPY src/ ./src/
COPY tsconfig.json ./
COPY --from=dashboard-builder /build/dist ./dashboard/dist
EXPOSE 3000
CMD ["bun", "src/index.ts"]
```

- [ ] **Step 2: Create `.dockerignore`**
